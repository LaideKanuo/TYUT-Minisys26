# 实验一：基于MIPS指令系统的单周期Minisys-1 CPU设计

## 一、总体架构

本设计是一个单周期 MIPS CPU，数据通路如下：

```
程序ROM(prgrom) → 取指(ifetc32) → 译码(idecode32) → 执行(executs32) → 访存(dmemory32+memorio)
                        ↑               ↑              ↑
                  控制器(control32) ←─ instruction[31:26],[5:0]
                                          │
                                   LED / Switch 外设
```

---

## 二、取指部件（ifetc32.v）

### 功能
产生下一条指令地址（PC），从 ROM 读取指令送往译码单元。

### 实现原理

**1. PC+4 计算**
```verilog
assign PC_plus_4[31:2] = PC[31:2] + 1'b1;
assign PC_plus_4[1:0]  = 2'b00;
```
由于指令是 32 位（4 字节）对齐的，PC+4 等效于高 30 位加 1，低 2 位恒为 0。

**2. 下一条 PC 选择（next_PC）**
按优先级判断：
- **JR**（Jrn=1）：`next_PC = Read_data_1`——跳转到寄存器值
- **BEQ**（Branch=1 且 Zero=1）：`next_PC = Add_result`——相等则跳转
- **BNE**（nBranch=1 且 Zero=0）：`next_PC = Add_result`——不等则跳转
- **J**（Jmp=1）：`next_PC = {PC_plus_4[31:28], Instruction[25:0], 2'b00}`——无条件跳转
- **JAL**（Jal=1）：同上，同时保存返回地址
- **默认**：`next_PC = PC_plus_4`——顺序执行

**3. PC 寄存器**
在时钟下降沿更新 PC，同时锁存 PC+4（供 JAL 指令将返回地址写入 $ra 寄存器）。

---

## 三、译码部件（idecode32.v）

### 功能
解析指令各字段，读取寄存器，处理立即数符号扩展，选择写回数据。

### 实现原理

**1. 指令字段提取**
| 字段 | 位 | 含义 |
|------|-----|------|
| rs | instruction[25:21] | 源寄存器1 |
| rt | instruction[20:16] | 源寄存器2 / I型目标寄存器 |
| rd | instruction[15:11] | R型目标寄存器 |
| immediate | instruction[15:0] | 16位立即数 |
| opcode | instruction[31:26] | 操作码 |

**2. 寄存器堆**
32 个 32 位寄存器，`$0` 恒为 0。读操作是组合逻辑（直接索引），写操作在时钟上升沿且 RegWrite=1 时写入。

**3. 符号扩展**
```verilog
assign Sign_extend = {{16{Instruction[15]}}, Instruction[15:0]};
```
将 16 位立即数符号扩展到 32 位。

**4. 写回数据选择（多路复用）**
- JAL：写回 `opcplus4`（返回地址 PC+4）
- LW/MemtoReg：写回内存读出的数据
- 其他：写回 ALU 计算结果

---

## 四、控制器（control32.v）

### 功能
根据指令 opcode 和 function 码生成 13 个控制信号。

### 控制信号与指令对应关系

| 信号 | 含义 | 为1的指令 |
|------|------|----------|
| RegDST | 目标寄存器选 rd(1) 还是 rt(0) | R型指令 |
| ALUSrc | 第二操作数选立即数(1) 还是寄存器(0) | I型、LW、SW |
| MemtoReg | 写回数据来自内存(1) 还是ALU(0) | LW |
| RegWrite | 是否写寄存器 | R型、I型、LW、JAL |
| MemWrite | 是否写内存 | SW |
| Branch | 是否BEQ指令 | BEQ |
| nBranch | 是否BNE指令 | BNE |
| Jmp | 是否J指令 | J |
| Jal | 是否JAL指令 | JAL |
| Jrn | 是否JR指令 | JR(function=001000) |
| I_format | 是否纯I型ALU指令 | ADDI等(opcode[5:3]=001) |
| Sftmd | 是否移位指令 | SLL/SRL/SRA等 |
| ALUOp | ALU操作编码 | bit1=R或I型, bit0=分支类 |

**译码逻辑**：R格式（opcode=000000）、LW（100011）、SW（101011）、BEQ（000100）、BNE（000101）、J（000010）、JAL（000011）、I型（opcode[5:3]=001）分别通过 opcode 比较产生。

---

## 五、执行单元（executs32.v）

### 功能
执行 ALU 运算（加减、逻辑、比较、移位）。

### 实现原理

**1. 操作数选择**
- A 输入：始终来自 `Read_data_1`（rs 值）
- B 输入：`ALUSrc=0` 选 `Read_data_2`（rt 值），`ALUSrc=1` 选立即数

**2. ALU 控制编码（ALU_ctl）**
根据 Function_opcode 和 ALUOp 生成 3 位 ALU 控制码：

| ALU_ctl | 运算 | 对应指令 |
|---------|------|----------|
| 000 | AND | and, andi |
| 001 | OR | or, ori |
| 010 | ADD | add, addi, lw, sw |
| 011 | ADDU | addu |
| 100 | XOR | xor |
| 101 | NOR | nor |
| 110 | SUB | sub, beq, bne |
| 111 | SLT/SUBU | slt, slti, subu |

**3. 移位操作**
由 Sftmd 信号控制，根据 Function_opcode[2:0] 选择移位类型：
- SLL/SRL/SRA：按 shamt（立即数）移位
- SLLV/SRLV/SRAV：按 rs 寄存器值移位

**4. Zero 标志**
`Zero = (ALU_output_mux == 0) ? 1 : 0`，供 BEQ/BNE 使用。

**5. 分支地址计算**
`Add_Result = PC_plus_4 + Sign_extend << 2`，计算 BEQ/BNE 的目标地址。

## 六、存储器设计

### dmemory32.v
数据 RAM 模块，封装 Block Memory Generator IP。CPU 正常工作时使用 CPU 时钟，下载程序时切换到 UART 时钟。

### memorio.v
存储器/IO 地址译码和路由：
- 对地址 `0xFFFFFC60~0xFFFFFC6F` 的操作路由到 LED
- 对地址 `0xFFFFFC70~0xFFFFFC7F` 的操作路由到 Switch
- 其他地址路由到数据 RAM
- 读数据来源选择：`memread=1` 时从 RAM 读取，否则从 IO 读取

---

## 七、外设模块

### leds.v
24 位 LED 输出。通过 `ledaddr[1:0]` 分两次写入（低 16 位 + 高 8 位）。

### switchs.v
24 位开关输入。通过 `switchaddr[1:0]` 分两次读取（低 16 位 + 高 8 位）。

### ioread.v
I/O 读数据多路切换，当前仅连接了 Switch 的读通道。

---

## 八、仿真验证

仿真测试程序（cputest.asm）覆盖了以下指令：
- **逻辑立即数**：ori（31条，初始化全部寄存器）
- **访存**：lw、sw
- **算术**：add、sub、subu
- **逻辑**：and、or、xor、nor
- **比较**：slt
- **分支**：beq、bne
- **跳转**：j、jal、jr
- **立即数运算**：addi
- **移位**：sll、srl、srlv、sra
- **加载立即数**：lui

仿真结果：PC 正常递增，指令正确取指译码执行，ALU 计算结果正确，分支跳转目标地址正确，寄存器正确读写。

### 实验二需要在实验一基础上做什么

根据指导书第 3.3 节——「Minisys-1 CPU 的简单接口设计」：

**（1）控制器扩展**：将 `control32.v` 升级为 `controlIO32.v`
- 新增输入：`Alu_resultHigh[21:0]`（用于区分 RAM 访问和 I/O 访问）
- 新增输出：`MemRead`（存储器读）、`IORead`（I/O 读）、`IOWrite`（I/O 写）
- `MemtoReg` 改为 `MemorIOtoReg`（存储器或 I/O 数据写回寄存器）
- `RegWrite` 逻辑修改：增加排除 JR 指令的条件

**（2）数码管模块添加**：新建数码管驱动模块（如 `digitube.v`）
- 两个 4 位七段数码管，动态扫描显示
- 基地址 `0xFFFFFC80`（根据表 3-22 接口基地址分配）
- 需要段选信号和位选信号控制

**（3）LED 完善**：leds.v 已基本完成，需确认与扩展后 memorio 的接口匹配

**（4）memorio.v 扩展**：增加数码管的片选译码（新增 DigitubeCtrl 信号）

**（5）ioread.v 扩展**：增加数码管的读数据通道（如需要回读显示内容）

**（6）顶层 minisys.v 更新**：
- 替换 control32 为 controlIO32
- 添加数码管模块实例化
- 连接新增控制信号
