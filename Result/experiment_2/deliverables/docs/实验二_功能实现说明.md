## 二、控制器扩展（control32.v）

### 2.1 变更内容

实验一的控制器的 `MemtoReg` 信号只区分"内存"和"ALU 结果"两种写回来源。实验二需要区分内存访问和 IO 访问（因为两者在地址空间中重叠但物理设备不同）。

**新增端口**：

| 端口名 | 方向 | 位宽 | 说明 |
|--------|------|------|------|
| Alu_resultHigh | input | [21:0] | ALU 结果的高 22 位，用于判断访问目标是 RAM 还是 IO |
| MemorIOtoReg | output | 1 | 替换原 MemtoReg，指示写回数据来自外部（内存或 IO） |
| MemRead | output | 1 | 读内存（非 IO） |
| MemWrite | output | 1 | 写内存（非 IO） |
| IORead | output | 1 | 读 IO |
| IOWrite | output | 1 | 写 IO |

**IO 地址译码核心逻辑**：

```verilog
wire is_io = (Alu_resultHigh[21:0] == 22'h3FFFFF);
```

当 `alu_result[31:10]` 的高 22 位全为 1 时（即地址在 `0xFFFFFC00` 以上），判定为 IO 空间访问。

**访存信号拆分**：

```verilog
MemRead  = Lw && !is_io;   // LW 访问 RAM
IORead   = Lw &&  is_io;   // LW 访问 IO
MemWrite = Sw && !is_io;   // SW 访问 RAM
IOWrite  = Sw &&  is_io;   // SW 访问 IO
```

### 2.2 同时修复的 Bug

- **Bug #8（JR 指令 RegWrite 未排除）**：`RegWrite = ((R_format && !Jrn) || I_format || Lw || Jal) ? 1 : 0`，JR 指令不再错误写寄存器

## 三、数码管模块添加（digitube.v）

### 3.1 设计方案

- **基地址**：`0xFFFFFC80`（IO 地址空间，与 LED/Switch 相邻）
- **显示原理**：两个 4 位七段数码管，动态扫描显示。由于两数码管共享段选信号线，必须交替点亮
- **扫描频率**：23MHz 时钟 ÷ 2^13 ≈ 2.8kHz（>1kHz，肉眼无闪烁感）

### 3.2 模块接口

| 端口 | 方向 | 位宽 | 说明 |
|------|------|------|------|
| clk | input | 1 | 系统时钟 23MHz |
| rst | input | 1 | 复位 |
| digitube_cs | input | 1 | 片选（来自 memorio，地址 0xFFFFFC8x 时=1） |
| digitube_write | input | 1 | 写使能（来自 control32.IOWrite） |
| digitube_addr | input | [1:0] | 地址低 2 位 |
| write_data | input | [15:0] | 写入数据（低 8 位有效：digit1[3:0] + digit0[3:0]） |
| digitube_rdata | output | [15:0] | 回读数据 |
| seg | output | [7:0] | 段选信号 {dp, g, f, e, d, c, b, a}，共阳极（0=亮） |
| an | output | [3:0] | 位选信号，低有效（0=选通） |

### 3.3 写入寄存器格式

```
write_data[7:4] → digit1（左数码管，显示高位）
write_data[3:0] → digit0（右数码管，显示低位）
```

### 3.4 段码译码表（共阳极）

| 4-bit 值 | 显示 | seg[7:0] (dp,g,f,e,d,c,b,a) |
|----------|------|---------------------------|
| 0 | 0 | 11000000 |
| 1 | 1 | 11111001 |
| 2 | 2 | 10100100 |
| 3 | 3 | 10110000 |
| 4 | 4 | 10011001 |
| 5 | 5 | 10010010 |
| 6 | 6 | 10000010 |
| 7 | 7 | 11111000 |
| 8 | 8 | 10000000 |
| 9 | 9 | 10010000 |
| A | A | 10001000 |
| b | b | 10000011 |
| C | C | 11000110 |
| d | d | 10100001 |
| E | E | 10000110 |
| F | F | 10001110 |

3.5 注意事项

- 当前段码按**共阳极**（0=亮, 1=灭）编写。如果 Minisys 板实际使用共阴极，需将所有段码值按位取反
- 位选信号 an[3:0] 低有效：`1110`=digit0 选通，`1101`=digit1 选通

## 四、IO 路由扩展

### 4.1 memorio.v 变更

新增 `DigitubeCtrl` 输出端口：

```verilog
assign DigitubeCtrl = ((iorw==1) && (caddress[31:4] == 28'hFFFFFC8)) ? 1'b1 : 1'b0;
```

### 4.2 ioread.v 变更

- 新增 `digitubectrl` 输入和 `ioread_data_digitube` 输入
- 新增数码管读数据通道：`else if (digitubectrl == 1) ioread_data = ioread_data_digitube`
- **修复 latch**：所有 else 分支赋确定值 `16'b0` 而非自赋值 `ioread_data = ioread_data`

### 4.3 IO 地址空间完整映射

| 地址范围 | 外设 | 片选信号 | 位宽 | 读写 |
|---------|------|---------|------|------|
| 0x00000000 ~ 0x0000FFFF | RAM（64KB） | — | 32bit | R/W |
| 0xFFFFFC60 ~ 0xFFFFFC6F | LED | LEDCtrl | 24bit | W |
| 0xFFFFFC70 ~ 0xFFFFFC7F | 拨码开关 | SwitchCtrl | 24bit | R |
| 0xFFFFFC80 ~ 0xFFFFFC8F | 数码管 | DigitubeCtrl | 16bit(2×8) | W/R |

## 五、顶层集成（minisys.v）

### 5.1 关键信号连接

| 信号 | 来源 | 去向 | 说明 |
|------|------|------|------|
| `alu_result_high[21:0]` | alu_result[31:10] | control32.Alu_resultHigh | IO 地址译码依据 |
| `memread` | control32.MemRead | memorio.memread | 内存读 |
| `ioread` | control32.IORead | memorio.ioread, ioread.ior | IO 读 |
| `iowrite` | control32.IOWrite | memorio.iowrite, leds, digitube | IO 写 |
| `memoriotoreg` | control32.MemorIOtoReg | idecode32.MemtoReg | 写回数据选择 |
| `digitubectrl` | memorio.DigitubeCtrl | digitube.digitube_cs, ioread.digitubectrl | 数码管片选 |

### 5.2 实验一 workaround 移除

实验一中 `assign memread = memreg;`（利用 MemtoReg 信号间接驱动 memread）**已被移除**。实验二中 `memread` 由 control32 的 `.MemRead` 端口直接驱动，信号语义清晰。
